# 战略分析技能 (strategic-analysis)

围绕特定战略问题进行需求理解、逻辑拆解storyline，通过案头研究、专家访谈等形式给出独特战略判断，形成科技公司偏好的冷静深度PPT。

## 功能简介

- **需求理解**：确保准确理解用户的战略问题，有歧义时先确认
- **逻辑拆解**：将复杂问题拆解为可研究的子问题（议题树）
- **证据驱动**：每个判断都要有调研的论据、事实、数据支撑
- **独特视角**：基于数据和逻辑形成独特战略判断，避免泛泛而谈
- **冷静深度**：采用科技公司偏好的理性、数据驱动、深度分析风格

## 适用场景

当用户需要做以下类型分析时使用此技能：

- **行业研究**：市场规模、增长趋势、竞争格局
- **市场调研**：用户画像、需求分析、机会评估
- **竞品分析**：竞对能力、定位差异、市场份额对比
- **商业决策**：商业模式、战略选择、投资并购
- **产品/技术路线**：产品规划、技术栈选择、发展路径
- **企业战略**：增长策略、转型策略、竞争应对

## 不适用场景

- 需要实时交易数据的分析
- 需要访问私有数据（除非用户提供）
- 需要预测未来的具体数值
- 需要法律、财务等专业领域的专业判断

## 使用方式

### 快速开始

1. 触发技能：在对话中输入"战略分析"或相关关键词
2. 提供战略问题：描述你想解决的核心问题
3. 确认议题树：确认需要分析的子问题
4. 等待研究：AI进行案头研究和深度分析
5. 获取输出：获得Storyline和对应的PPT内容

### 输入格式

```
[核心问题] + [可选补充信息]

示例1：
"百度AI模型商业化应该如何制定战略？"

示例2：
"Coding Plan市场竞争格局如何？我们有机会吗？重点分析字节和DeepSeek"

示例3：
"分析AI编程助手市场，给出我们的战略选择建议"
```

### 输出格式

技能输出两类文档：

#### Storyline

- 核心判断（1-2个）
- 关键论据（3-5个）
- 支撑数据（多个）
- 建议/决策建议

#### PPT

- 每页标题是观点
- 正文是各种途径调研的论据、事实、数据
- 采用科技公司偏好的冷静深度风格

### 输出示例

**Storyline输出示例：**
```markdown
# 具身智能竞争战略分析

## 核心判断
1. 具身智能市场仍处于早期阶段，竞争格局尚未稳定
2. 不同厂商在技术路径和落地场景上存在明显差异

## 关键论据

2.1 行业发展阶段分析
> 具身智能仍处于从技术验证向规模化应用过渡阶段

支撑数据：
- 人形机器人出货量逐步增长（2026）- 行业报告
- 多数应用仍集中在试点或小规模部署 - 公开资料
- 商业化案例数量有限 - 行业调研
...

---

2.2 技术路径差异
> 不同厂商在感知、控制与决策等方面采用不同技术方案

支撑数据：
- 多种技术路线并行发展 - 技术资料
- 不同方案在成本与性能上差异明显 - 行业分析
- 技术标准尚未统一 - 行业观察
...

---

2.3 应用场景探索
> 具身智能在工业与服务等场景中逐步落地

支撑数据：
- 工业场景为主要应用方向 - 行业报告
- 服务场景处于探索阶段 - 调研数据
- 不同场景对能力要求差异较大 - 案例分析
...
```

**PPT输出示例：**
```
第1页：核心判断
标题：具身智能市场仍处于早期阶段，竞争格局尚未稳定
论据：行业处于技术验证向规模化应用过渡阶段
数据：[市场发展阶段图]

第2页：竞争格局
标题：不同厂商在技术路径与应用场景上存在差异
论据：技术路线多样，应用场景仍在探索
数据：[竞争态势图]
```

## 依赖要求

- **数据搜索能力**：需要联网搜索功能
- **文件读取能力**：需要读取参考文档和模板
- **分析时长**：复杂战略分析可能需要多次交互，无法一次性完成

## 质量标准

输出需要满足以下标准：

- **D1**: 核心判断清晰且有洞察
- **D2**: 论据充分且来源可靠
- **D3**: 逻辑自洽且结构清晰
- **D4**: 数据准确且有验证
- **D5**: 观点独特且有价值
- **D6**: 表达精准且符合风格
- **D7**: 无重大事实错误

详细标准请参考 `references/rubric.md`。
