# 分析框架

本文档提供战略分析常用的分析框架，帮助系统化思考和分析。

## 一、SWOT分析

### 用途
评估企业或战略的内外部环境。

### 框架

| 内部 | 优势 (Strengths) | 劣势 (Weaknesses) |
|-----|-----------------|------------------|
| | - 核心能力 | - 能力短板 |
| | - 独特资源 | - 资源约束 |
| | - 市场地位 | - 成本劣势 |
| 外部 | 机会 (Opportunities) | 威胁 (Threats) |
| | - 市场增长 | - 竞争加剧 |
| | - 技术变革 | - 政策风险 |
| | - 政策支持 | - 替代品威胁 |

### 应用要点

- 优势要相对竞对而言，劣势要具体可改善
- 机会要可把握，威胁要可应对
- 形成战略：SO（利用优势把握机会）、WO（克服劣势把握机会）等

---

## 二、波特五力模型

### 用途
评估行业竞争态势和盈利潜力。

### 框架

| 力量 | 说明 | 分析要点 |
|-----|------|---------|
| **供应商议价能力** | 供应商影响行业的能力 | 供应商集中度、替代供应商可获性、切换成本 |
| **买方议价能力** | 买方影响行业的能力 | 买方集中度、产品差异化、买方信息透明度 |
| **新进入者威胁** | 新竞争者进入的可能性 | 进入壁垒、规模经济、品牌认知 |
| **替代品威胁** | 其他产品/服务的替代可能性 | 替代品价格性能、切换成本 |
| **现有竞争者竞争** | 现有竞争者的竞争激烈程度 | 竞争者数量、行业增长率、退出壁垒 |

### 应用要点

- 各力量强弱决定行业盈利潜力
- 力量随时间变化，需动态评估
- 识别可改变各力量的杠杆点

---

## 三、价值链分析

### 用途
分析价值创造环节，识别优化机会。

### 框架

```
研发 → 采购 → 生产 → 营销 → 销售 → 服务
  ↓      ↓      ↓      ↓      ↓      ↓
价值  价值   价值   价值   价值   价值
```

### 应用要点

- 识别哪些环节创造价值，哪些是成本
- 分析各环节与竞对的差异
- 寻找外包或整合的机会

---

## 四、增长矩阵（Ansoff矩阵）

### 用途
评估增长机会，选择增长路径。

### 框架

| | 现有市场 | 新市场 |
|-----|---------|--------|
| **现有产品** | 市场渗透 | 市场开发 |
| **新产品** | 产品开发 | 多元化 |

### 应用要点

- 市场渗透：提升现有产品在现有市场的份额
- 市场开发：现有产品进入新市场
- 产品开发：新产品进入现有市场
- 多元化：新产品进入新市场（风险最高）

---

## 五、客户旅程分析

### 用途
理解用户体验，识别优化机会。

### 框架

```
认知 → 考虑 → 评估 → 购买 → 使用 → 忠诚
  ↓      ↓      ↓      ↓      ↓      ↓
痛点  痛点  痛点  痛点  痛点  痛点
机会  机会  机会  机会  机会  机会
```

### 应用要点

- 识别每个阶段的用户痛点和机会
- 关注用户流失的关键节点
- 考虑不同用户类型的旅程差异

---

## 六、3C模型

### 用途
分析公司、客户、竞争者三者关系。

### 框架

| 维度 | 分析问题 |
|-----|---------|
| **公司 (Company)** | 我们的能力、资源、战略是什么？ |
| **客户 (Customer)** | 客户需求、行为、决策因素是什么？ |
| **竞争者 (Competitor)** | 竞对的能力、战略、差异化是什么？ |

### 应用要点

- 识别公司的独特定位
- 理解客户的真实需求
- 分析竞对的威胁和机会

---

## 七、技术采用曲线

### 用途
评估技术成熟度和市场时机。

### 框架

```
创新者 → 早期采用者 → 早期大众 → 晚期大众 → 落后者
  2.5%     13.5%       34%       34%      16%
```

### 应用要点

- 识别技术所处阶段
- 判断市场时机是否成熟
- 制定相应的市场策略

---

## 八、PEST分析

### 用途
评估宏观环境对企业的影响。

### 框架

| 维度 | 分析要点 |
|-----|---------|
| **政治 (Political)** | 政策、法规、政治稳定性 |
| **经济 (Economic)** | 经济增长、利率、通胀、汇率 |
| **社会 (Social)** | 人口结构、文化、生活方式 |
| **技术 (Technological)** | 技术变革、专利、创新速度 |

### 应用要点

- 识别外部机会和威胁
- 评估宏观环境的变化趋势
- 考虑不同地区的PEST差异

---

## 九、核心能力分析

### 用途
识别企业的核心竞争力和可持续优势。

### 框架

| 维度 | 评估问题 |
|-----|---------|
| **价值性** | 能为客户创造独特价值吗？ |
| **稀缺性** | 竞对是否难以复制？ |
| **难以模仿** | 模仿的成本和时间有多高？ |
| **组织性** | 是否融入组织体系？ |

### 应用要点

- 区分核心能力与一般能力
- 核心能力要满足VRIN（价值、稀缺、难模仿、组织）
- 核心能力要可延伸到新业务

---

## 十、场景分析

### 用途
评估不同未来情景下的战略应对。

### 框架

```
情景1（乐观）  →  战略1
情景2（基准）  →  战略2
情景3（悲观）  →  战略3
情景4（破坏）  →  战略4
```

### 应用要点

- 识别关键不确定性变量
- 构建多种可能的未来情景
- 评估每种情景下的最优战略
- 制定适应性强的战略

---

## 框架选择指南

根据研究问题选择合适的框架：

| 研究问题 | 推荐框架 |
|---------|---------|
| 评估行业竞争 | 波特五力 |
| 评估企业定位 | SWOT、3C |
| 评估增长机会 | 增长矩阵 |
| 评估技术时机 | 技术采用曲线 |
| 评估宏观环境 | PEST |
| 评估客户体验 | 客户旅程 |
| 评估价值创造 | 价值链 |
| 评估不确定性 | 场景分析 |
| 评估核心竞争力 | 核心能力分析 |

---

## 框架使用原则

1. **组合使用**：一个研究通常需要多个框架组合
2. **因地制宜**：根据具体问题调整框架
3. **数据驱动**：框架是工具，结论要基于数据
4. **逻辑自洽**：不同框架的分析结果要逻辑一致
5. **洞察导向**：框架是手段，洞察才是目标
