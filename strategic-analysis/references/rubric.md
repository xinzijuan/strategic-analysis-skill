# 质量评估标准（Rubric）

本文档定义战略分析输出的质量评估标准，用于自我检查和质量控制。

## D1: 核心判断清晰且有洞察

### 含义
核心判断要清晰明确，有真正的洞察，而非泛泛而谈的事实罗列。

### 评分标准

| 分数 | 标准 |
|-----|------|
| 5分 | 核心判断清晰，有独特洞察，能改变认知或启发思考 |
| 4分 | 核心判断清晰，有一定洞察，超出常识 |
| 3分 | 核心判断清晰，但洞察一般 |
| 2分 | 核心判断存在，但不够清晰或洞察有限 |
| 1分 | 没有核心判断，或判断模糊不清 |

### 红线（不可接受）
- 没有明确的核心判断
- 核心判断自相矛盾

---

## D2: 论据充分且来源可靠

### 含义
核心判断要有充分的论据支撑，论据来自可靠来源并明确标注。

### 评分标准

| 分数 | 标准 |
|-----|------|
| 5分 | 论据充分，来源权威，多源交叉验证 |
| 4分 | 论据充分，来源可靠，有主要来源标注 |
| 3分 | 论据基本充分，有来源但不够详细 |
| 2分 | 论据不足，或来源不够可靠 |
| 1分 | 没有论据或没有来源 |

### 红线（不可接受）
- 关键判断没有论据支撑
- 论据来源完全不可信
- 使用捏造或编造的数据

---

## D3: 逻辑自洽且结构清晰

### 含义
分析要逻辑自洽，结构清晰，各部分之间有逻辑关系。

### 评分标准

| 分数 | 标准 |
|-----|------|
| 5分 | 逻辑严密，结构清晰，层次分明，各部分逻辑关系明确 |
| 4分 | 逻辑基本自洽，结构清晰，有层次感 |
| 3分 | 逻辑基本自洽，结构尚可，层次一般 |
| 2分 | 逻辑有瑕疵，结构混乱，层次不清 |
| 1分 | 逻辑矛盾，结构混乱 |

### 红线（不可接受）
- 存在明显的逻辑矛盾
- 结构完全混乱，无法理解

---

## D4: 数据准确且有验证

### 含义
使用的数据要准确，经过验证，标注数据来源和时间。

### 评分标准

| 分数 | 标准 |
|-----|------|
| 5分 | 数据准确，有验证，来源和时间标注完整 |
| 4分 | 数据准确，有验证，有来源标注 |
| 3分 | 数据基本准确，有来源但验证不足 |
| 2分 | 数据有明显错误或矛盾 |
| 1分 | 数据错误严重或没有来源 |

### 红线（不可接受）
- 关键数据错误
- 使用明显过时的数据（除非有说明）
- 捏造数据

---

## D5: 观点独特且有价值

### 含义
分析要有独特观点，能提供新视角或新认知，而非重复常识或竞对观点。

### 评分标准

| 分数 | 标准 |
|-----|------|
| 5分 | 观点独特且有高价值，能启发新思考或改变决策 |
| 4分 | 观点较独特，有一定价值 |
| 3分 | 观点有一定独特性，但价值一般 |
| 2分 | 观点基本独特，但价值有限 |
| 1分 | 没有独特观点，或观点无价值 |

### 红线（不可接受）
- 完全重复已知观点，没有任何独特性
- 观点完全没有价值

---

## D6: 表达精准且符合风格

### 含义
表达要精准，避免模糊表述，符合科技公司偏好的冷静深度风格。

### 评分标准

| 分数 | 标准 |
|-----|------|
| 5分 | 表达精准，符合风格，语言简洁有力 |
| 4分 | 表达较精准，基本符合风格 |
| 3分 | 表达尚可，风格基本符合 |
| 2分 | 表达不够精准，风格不够符合 |
| 1分 | 表达模糊，风格不符 |

### 红线（不可接受）
- 关键表述模糊到无法理解
- 风格完全不符合要求（过于情绪化、口语化等）

---

## D7: 无重大事实错误

### 含义
不能有重大的事实错误，特别是影响结论的关键事实。

### 评分标准

| 分数 | 标准 |
|-----|------|
| 5分 | 无事实错误 |
| 4分 | 无重大事实错误，有轻微瑕疵 |
| 3分 | 有一般事实错误但不影响结论 |
| 2分 | 有重大事实错误，结论受影响 |
| 1分 | 有多处重大事实错误 |

### 红线（不可接受）
- 有关键事实错误，完全误导结论
- 基本事实完全错误

---

## 综合评分

### 计算方式
总分 = (D1 + D2 + D3 + D4 + D5 + D6 + D7) / 7

### 等级划分

| 总分 | 等级 | 说明 |
|-----|------|------|
| 4.5-5.0 | 优秀 | 可以直接输出 |
| 4.0-4.4 | 良好 | 可输出，有少量改进空间 |
| 3.5-3.9 | 合格 | 可输出，有明显改进空间 |
| 3.0-3.4 | 勉强 | 需要修改后再输出 |
| <3.0 | 不合格 | 需要重新分析和撰写 |

### 红线处理

- 如果触犯任何红线，必须修改后才能输出
- 红线是"否决项"，即使总分很高也不能输出

---

## 自检流程

输出前按以下流程自检：

1. **检查D1**：核心判断清晰吗？有洞察吗？
2. **检查D2**：论据充分吗？来源可靠吗？
3. **检查D3**：逻辑自洽吗？结构清晰吗？
4. **检查D4**：数据准确吗？有验证吗？
5. **检查D5**：观点独特吗？有价值吗？
6. **检查D6**：表达精准吗？风格符合吗？
7. **检查D7**：有重大事实错误吗？
8. **检查红线**：是否触犯任何红线？
9. **计算总分**：总分是否达到3.5以上？
10. **决定输出**：达到标准输出，否则修改

---

## 用户反馈整合

用户反馈后，按以下方式整合：

| 反馈类型 | 处理方式 |
|---------|---------|
| 事实错误 | 修正后自检D7和D4 |
| 逻辑问题 | 修正后自检D3 |
| 论据不足 | 补充论据后自检D2 |
| 表达问题 | 修正后自检D6 |
| 观点问题 | 重新分析后自检D1、D5 |
| 风格问题 | 调整后自检D6 |

反馈整合后在 `progress.md` 中记录修改内容。
