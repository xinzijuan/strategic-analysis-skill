# 研究方法论

本文档说明战略分析中使用的研究方法，确保研究的系统性和可信度。

## 研究流程

### 1. 问题定义

- 明确研究问题的边界和范围
- 识别关键变量和指标
- 确定研究的深度和时间范围

### 2. 信息收集

#### 数据来源类型

| 类型 | 可信度 | 使用场景 |
|-----|--------|---------|
| 权威机构报告 | 高 | 行业整体数据、趋势判断 |
| 公司财报/公告 | 高 | 公司经营数据、战略方向 |
| 学术研究 | 中高 | 理论框架、验证研究 |
| 行业媒体报道 | 中 | 最新动态、案例分析 |
| 社交媒体/用户评论 | 低 | 用户感知、市场情绪 |

#### 数据收集原则

- 优先使用可信度高的数据源
- 多源交叉验证关键数据
- 记录数据来源和获取时间
- 识别数据的时效性和适用范围

### 3. 数据验证

#### 验证方法

| 方法 | 说明 | 适用场景 |
|-----|------|---------|
| 横向验证 | 对比不同来源的数据 | 关键指标、规模数据 |
| 纵向验证 | 对比同一来源的历史数据 | 趋势判断 |
| 逻辑验证 | 检查数据是否符合逻辑常识 | 异常值识别 |
| 专家验证 | 引用专家观点或数据 | 专业领域判断 |

#### 矛盾数据处理

- 记录矛盾数据及其来源
- 分析矛盾产生的原因（统计口径、时间点、来源可信度）
- 选择最可信的数据或注明差异
- 在分析中说明数据不确定性的影响

### 4. 分析框架

#### 常用分析维度

| 维度 | 说明 | 关键问题 |
|-----|------|---------|
| 市场分析 | 市场规模、增长率、结构 | 市场有多大？在增长吗？ |
| 竞争分析 | 竞对数量、定位、能力 | 谁是竞对？他们有什么优势？ |
| 客户分析 | 客户画像、需求、行为 | 客户是谁？他们需要什么？ |
| 产品分析 | 产品功能、体验、差异化 | 产品有什么特点？差异化在哪？ |
| 商业模式分析 | 价值主张、盈利模式、成本结构 | 怎么创造价值？怎么赚钱？ |
| 能力分析 | 技术能力、渠道能力、组织能力 | 我们有什么能力？缺什么？ |

#### 分析框架选择

根据研究问题选择合适的分析框架：

- SWOT分析：评估内外部环境
- 波特五力：评估行业竞争态势
- 价值链分析：评估价值创造环节
- 客户旅程分析：评估用户体验
- 增长矩阵：评估业务增长机会

### 5. 结论形成

#### 结论形成的原则

1. **证据驱动**：每个结论都要有数据或论据支撑
2. **逻辑自洽**：结论之间要逻辑一致，不能矛盾
3. **洞察导向**：结论要有洞察，不能是事实的简单罗列
4. **可行动性**：结论要有可操作的建议或决策含义

#### 结论验证

- 检查是否有反例或例外情况
- 考虑不同假设下的结论变化
- 评估结论的置信度和不确定性
- 明确结论的适用范围和边界

### 6. 输出标准

| 标准 | 要求 |
|-----|------|
| 清晰性 | 结论和论据表述清晰，无歧义 |
| 准确性 | 数据准确，来源明确 |
| 相关性 | 论据与结论高度相关 |
| 完整性 | 覆盖关键问题和维度 |
| 洞察性 | 有独特见解，非泛泛而谈 |
| 可信性 | 数据来源可靠，逻辑严密 |

## 常见研究陷阱

### 陷阱1：确认偏差

- **现象**：只收集支持预设观点的数据
- **避免**：主动寻找反例，考虑多种可能性

### 陷阱2：幸存者偏差

- **现象**：只关注成功案例，忽视失败案例
- **避免**：分析成功和失败两类案例

### 陷阱3：因果倒置

- **现象**：把相关关系误认为因果关系
- **避免**：通过逻辑和实验验证因果关系

### 陷阱4：数据过时

- **现象**：使用过时数据得出当前结论
- **避免**：关注数据时效性，使用最新数据

### 陷阱5：样本偏差

- **现象**：样本不能代表整体
- **避免**：评估样本代表性，补充更多数据源

## 研究记录

在研究过程中记录以下信息：

- 研究问题
- 数据来源及获取时间
- 数据验证过程
- 分析方法
- 不确定性说明
- 后续研究建议

这些信息记录在 `progress.md` 中，确保研究的可追溯性和可重复性。
